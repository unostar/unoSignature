## Architecture

| Hook / Endpoint | What it does |
|---|---|
| `woocommerce_after_checkout_validation` | Creates/checks signing request, blocks checkout via `$errors->add()` |
| `woocommerce_after_checkout_form` | JS: modal + iframe + postMessage listener + polling |
| `POST /wp-json/firma/v1/webhook` | HMAC-SHA256 verified, handles `completed` + `certificate.generated` |
| `GET /wp-json/firma/v1/signing-completed-status` | Polling endpoint for frontend status check |

**Key decisions (do not change without reading history):**
- Declined unlock: `timestamps.declined_on` + 48h
- Manual continue button, no auto-submit on `signing.completed`
- Polling 25├Ś600ms after postMessage before closing modal
- `send_signing_email: false`
- All debug via `FIRMA_DEBUG` constant
- Webhook fail-closed if `FIRMA_WEBHOOK_SECRET` is empty
- Owner-copy email on `certificate.generated`: subject = `name [- template_description]`, body = request ID + user fields only, attachment = `name.ext` (ext from download URL, no sanitize_file_name)
- Users parsed from `/users` response as `results` array (confirmed via MCP)
